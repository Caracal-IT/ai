# ai

`ai` is an npm-style CLI that bootstraps a structured AI workspace for any project.
It reads the wizard layout directly from the repository `src/` folder and generates
`.github/` content from the selected items.

## Repository Structure

```
bin/          CLI entry point
cli/          Command modules (init, detect, wizard, generate, sync, update)
lib/          Shared utilities (fs, prompts, templates, project-type)
src/          Wizard source content (required + category/item folders)
test/         Automated tests
```

## Source Layout (`src/`)

- `src/required/` тЖТ always copied into `.github/`
- `src/<category>/<item>/` тЖТ wizard sections and selectable items
- Each item can contain:
  - `instructions/**`
  - `skills/**`
  - `agents/**`

Example:

```
src/
тФЬтФАтФА required/
тФВ   тФЬтФАтФА instructions/
тФВ   тФЬтФАтФА skills/
тФВ   тФФтФАтФА agents/
тФЬтФАтФА capabilities/
тФВ   тФЬтФАтФА authentication/
тФВ   тФВ   тФФтФАтФА skills/
тФВ   тФФтФАтФА ...
тФФтФАтФА language/
    тФЬтФАтФА go/

    тФФтФАтФА kotlin/
        тФФтФАтФА instructions/kotlin.best-practices.instructions.md
```

## Usage

```bash
# Initialise a new AI workspace (interactive wizard)
npx github:Caracal-IT/ai init

# Initialise in a specific directory
npx github:Caracal-IT/ai init ./my-project

# Regenerate .github/
npx github:Caracal-IT/ai generate

# Update selections (pre-marks installed items and recopies them)
npx github:Caracal-IT/ai update
```

### Windows 11 note for `npx github:...`

If `npx github:Caracal-IT/ai ...` fails with:

`ENOENT: no such file or directory, lstat 'C:\Users\<you>\AppData\Roaming\npm'`

create the missing npm roaming folder once, then rerun the command:

```powershell
New-Item -ItemType Directory -Force "$env:APPDATA\npm" | Out-Null
npx github:Caracal-IT/ai init ./my-project
```

## What `init` does

1. Detects project type.
2. Prepares `.github/` managed paths.
3. Records any existing `.github/` paths as `excluded` in `project.ai.json` (files stay untouched, and nested folders are tracked too).
4. Reads `src/` folder structure (no hard-coded wizard structure).
5. Prompts by category and lets users select items via checkbox controls.
6. Copies `src/required` and selected category items into `.github/`.
7. Compiles `.github/{instructions,skills,agents}` directly from `src/` selections.
8. Records generated `.github/` files and generated item folders in the `managed` list inside `project.ai.json`.
9. Cleans up old `.github/` ignore entries from `.gitignore` so every file in `.github/` is source-controlled.
10. Creates project `README.md`.

Required starter content includes a `feature-documentation` skill for creating
and updating `docs/.../*.feature.md` files with an inline default template.

## What `update` does

1. Reads existing `project.ai.json` (or falls back to legacy `.ai/project.ai.json`).
2. Opens the same category wizard with already-installed items pre-marked.
3. Re-copies all marked items, including items already copied before.
4. Rebuilds `.github/` from selected `src/` items.
5. Scans `.github/` for files that are neither in `managed` nor in `excluded` (including folder-based entries).
6. For each unknown file, prompts: **Delete it?**
   - **Yes** тЖТ deletes the file.
   - **No** тЖТ adds it to the `excluded` list so it is never prompted again.

## Generated Structure

```
<project>/
тФЬтФАтФА project.ai.json               тЖР config + managed/excluded tracking (root)
тФЬтФАтФА .github/                      тЖР Copilot context (fully source-controlled)
тФВ   тФЬтФАтФА instructions/
тФВ   тФЬтФАтФА skills/                   тЖР each skill is generated as <name>/SKILL.md
тФВ   тФФтФАтФА agents/
тФФтФАтФА README.md
```

### `project.ai.json` fields

| Field | Description |
|-------|-------------|
| `name` | Project name |
| `version` | Schema version (3) |
| `type` | Detected project type |
| `selections` | Wizard selections per category |
| `excluded` | `.github/` files or folders the tool must never touch (pre-existing or user-kept); folder entries are preferred when they cover all child files |
| `managed` | `.github/` files or folders owned and regenerated by the tool; folder entries are preferred when they cover all child files |

## Adding a custom AI item

1. Create a category (or reuse one): `src/<category>/`
2. Create a selectable item folder: `src/<category>/<item>/`
3. Add files under any of:
   - `src/<category>/<item>/instructions/`
   - `src/<category>/<item>/skills/`
   - `src/<category>/<item>/agents/`
4. Run `npx github:Caracal-IT/ai update` in your target project and select the item.

The wizard section and labels are derived from folder names automatically.
